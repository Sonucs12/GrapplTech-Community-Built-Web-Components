## Usage 💡
Each component is designed to be easily integrated into your web projects. Below is an example of how to use a hero section component:

**Example: Using the 'Akhul' Hero Section**

1.  Ensure you have the `index.html` and `style.css` files from `Hero-Section-Components/Akhul/` in your project directory.

2.  Include the HTML structure in your `<body>` tag:
    ```html
    <!DOCTYPE html>
    <html lang="en">
    <head>
      <meta charset="UTF-8">
      <meta name="viewport" content="width=device-width, initial-scale=1.0">
      <title>My Project</title>
      <!-- Link to your main styles or component styles -->
      <link rel="stylesheet" href="./path/to/Akhul/style.css">
    </head>
    <body>

      <!-- Include the hero section component -->
      <section id="hero-section-akhul">
        <h1>This is the hero section</h1>
      </section>

      <!-- Include other project content -->

    </body>
    </html>
    ```

3.  **Linking CSS:** Make sure to link the `style.css` file appropriately in your HTML's `<head>` section.

## Contributing 🤝
We welcome contributions from students and developers! Please follow these steps to contribute:

1.  **Fork the Repository:** Create a personal copy of the repository.
2.  **Clone Your Fork:** Download your forked repository to your local machine.
    ```bash
    git clone https://github.com/your-username/GrapplTech-Community-Built-Web-Components.git
    cd GrapplTech-Community-Built-Web-Components
    ```
3.  **Create a New Branch:** Isolate your work.
    ```bash
    git checkout -b your-full-name-category
    ```
    *(e.g., `git checkout -b john-doe-hero-section`)*
4.  **Add Your Component:** Create a new folder named after your contribution (e.g., `JohnDoeHeroSection`) within the appropriate component category (e.g., `Hero-Section-Components/`). Add your HTML, CSS, and JS files.
5.  **Commit Changes:** Stage and commit your work with a descriptive message.
    ```bash
    git add .
    git commit -m "Added [Category] template by [Your Full Name]"
    ```
6.  **Push to Your Fork:** Upload your branch to your repository.
    ```bash
    git push origin your-full-name-category
    ```
7.  **Create a Pull Request:** Submit your changes to the original repository.
